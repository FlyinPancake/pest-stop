# Pest Stop

## Repo Layout

- `backend/`: Rust HTTP service for compact transit responses.
- `watchapp/`: Native C Pebble watchapp plus `PebbleKit JS` phone bridge.
- `docs/architecture.md`: system shape and next milestones.

## What Exists Today

- A Rust `axum` server with demo endpoints for health, stop search, and departures.
- A native C Pebble watchapp scaffold with a simple transit-style UI.
- A PebbleKit JS bridge scaffold in `watchapp/src/pkjs/index.js`.

## Backend Quickstart

```bash
cargo run -p bkk-server
```

The server starts on `127.0.0.1:3000` by default.

Useful endpoints:

- `http://127.0.0.1:3000/health`
- `http://127.0.0.1:3000/api/v1/stops/search?q=deak`
- `http://127.0.0.1:3000/api/v1/stops/demo/departures?limit=2`

## Watchapp Quickstart

```bash
cd watchapp
pebble build
```

To upload to an emulator:

```bash
pebble install --emulator basalt --logs build/watchapp.pbw
```

The watchapp is now built with the native Pebble C toolchain generated by `pebble new-project`.

## Important Note About Networking

The watchapp does not fetch HTTP directly. `watchapp/src/pkjs/index.js` runs on the phone and calls the Rust backend, then forwards a tiny message payload to the watch.

Right now the JS bridge is only a stub. When we wire it back up to the backend, `127.0.0.1:3000` will be fine for an emulator setup running on the same machine. For a real phone, it will need to point at a host the phone can reach on your network.
